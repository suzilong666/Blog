## 层叠

[层叠](https://developer.mozilla.org/zh-CN/docs/Web/CSS/Guides/Cascade/Introduction)

### 层叠是什么？

CSS中的“层叠”（Cascade）是它最核心的机制，也是“CSS”这个名字中“C”（Cascading，层叠）的由来。简单来说，**层叠就是当多个CSS规则同时作用于同一个元素，并且这些规则中某些样式声明发生冲突时，决定最终采用哪一个样式的一套优先级规则。**

### 为什么需要层叠？

在开发网页时，一个HTML元素可能会被多个CSS规则选中。比如：

```css
/* 规则A：作用于所有段落 */
p {
  color: blue;
}

/* 规则B：作用于class为“note”的元素 */
.note {
  color: red;
}
```

```html
<p class="note">这是一段文字</p>
```

这个 p 元素同时满足了规则A和规则B，而且两条规则都试图设置 color 属性。这时候浏览器就需要一套**“裁判规则”**来决定到底显示蓝色还是红色。这套裁判规则就是层叠。

## 继承

[继承](https://developer.mozilla.org/zh-CN/docs/Web/CSS/Guides/Cascade/Inheritance)

继承主要发生在与**文本内容**表现相关的属性上，因为这些属性通常希望在整个文本块内保持一致；而与**布局、盒子模型**相关的属性则一般不被继承，否则会导致不可控的页面错乱。
