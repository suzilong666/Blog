# 原型与原型链

## 原型
原型是 JavaScript 中实现**继承**的核心机制。每个函数都有一个 prototype 属性（除了箭头函数），每个对象都有一个 __proto__ 属性，它们指向的那个对象，被称之为原型。

通过原型，对象可以共享方法和属性，从而实现了基于原型的继承，这也是 JavaScript 区别于传统基于类继承的语言的重要特征。

## 构造函数、实例、原型的关系

```js
function Animal(name) {
  this.name = name;
}

Animal.prototype.eat = function() {
  console.log(`${this.name} is eating`);
};

const dog = new Animal('Buddy');

// 关系图
console.log(dog.__proto__ === Animal.prototype); // true
console.log(dog.__proto__.constructor === Animal); // true
console.log(Animal.prototype.constructor === Animal); // true
console.log(Animal.prototype.__proto__ === Object.prototype); // true
console.log(Object.getPrototypeOf(Animal) === Function.prototype); // true
```

## 原型链

当你试图访问一个对象的属性时，如果该对象自身没有这个属性，JavaScript 就会沿着 __proto__ 这条链向上查找，直到找到该属性或到达原型链的末端（null），这种层层连接的查找路径就形成了**原型链**。
